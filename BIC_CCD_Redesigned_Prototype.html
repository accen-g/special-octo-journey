🔎 Key Findings

📉
Systemic Data Provider Failure — Genesis & DCRM
Detected: 16 Mar 2026 · Affects 18 KRIs across all regions
SLA breach rate increased from 71% to 83% month-over-month, driven primarily by data non-receipt from Genesis (5 KRIs) and DCRM (13 KRIs). This is a 3-month worsening trend that correlates with the Q1 system migration. Recommend escalation to platform engineering team and interim manual data submission process.
Root Cause: System Migration Impact: High Genesis DCRM
⏱️
L1 Approval Bottleneck — 33% Rework Rate
Pattern: Last 3 months · Affects C&A workflow
L1 reviewers are rejecting one-third of submissions, adding an average 4.2 days to cycle time. Root cause analysis shows 80% of rejections relate to incomplete evidence documentation. Recommend: standardized checklist template for submitters, and optional pre-check validation before L1 submission.
Bottleneck: L1 Stage Evidence Quality Process Improvement

Positive signal · Consistent performer
UK-K01 has met SLA for 11 of the last 12 months, making it the most reliable KRI in the framework. The Olympus source system has near-perfect data delivery. This KRI can serve as a benchmark for data provider SLA design with other source systems.
