### 2. ORA-01400 NULL Primary Key Fix
**File:** `backend/app/models/__init__.py`

**Root cause:** SQLAlchemy `autoincrement=True` does not emit Oracle `IDENTITY` DDL. When tables already exist, `create_all()` is a no-op, leaving columns without auto-increment. Oracle receives NULL on INSERT → ORA-01400.

**Fix applied:** Added `Identity` to SQLAlchemy imports; changed all 23 table PKs:
```python
# Import change
from sqlalchemy import (
    Column, Integer, String, DateTime, Float, Text, Boolean,
    ForeignKey, UniqueConstraint, Index, CheckConstraint, Date, Identity
)

# PK pattern — applied to every table
# Before:
region_id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
# After:
region_id: Mapped[int] = mapped_column(Integer, Identity(start=1), primary_key=True)
```

**Required one-time step:** Drop all existing Oracle tables with `CASCADE CONSTRAINTS`, then restart. `create_all()` will then emit `GENERATED BY DEFAULT AS IDENTITY` DDL correctly.

`GENERATED BY DEFAULT AS IDENTITY` (not `ALWAYS`) was chosen to allow manual inserts during seeding.
